## 1. Integrita tier obsahu — NOVÁ: 10/10

### Klíčová změna oproti předchozímu auditu

Byl zaveden `resolveTierFeatures(d: StoredContractData): TierFeatures` jako **jediný autoritativní zdroj pravdy** pro veškerou logiku balíčků.

```typescript
// lib/contracts.ts
export function resolveTierFeatures(d: StoredContractData): TierFeatures {
  const tier = String(d.tier ?? 'basic').toLowerCase() as Tier;
  const legacyPremium = Boolean(d.notaryUpsell); // fallback pro starší drafty
  const hasPremiumClauses = legacyPremium || tier === 'professional' || tier === 'complete';
  const hasCompletePages = tier === 'complete';
  const archiveDays = tier === 'complete' ? 30 : 7;
  return { hasPremiumClauses, hasCompletePages, archiveDays };
}
```

| Vlastnost | Hodnota |
|---|---|
| Funkce volaná v lib/contracts.ts | 15× (1× definice + 14× v builder funkcích) |
| `hasPremiumClauses` použití | 59× |
| `hasCompletePages` použití | 3× (contracts + pdf + pdf) |
| `d.notaryUpsell` přímé použití | 2× (záměrně: definice typu + legacyPremium) |
| Přímé `d.notaryUpsell` v business logice | **0×** ✅ |

### Opraven kritický bug: work_contract `[object Object]`

Před opravou: `...premiumContent` (pole `ContractSection[]`) bylo rozptýleno do `body: string[]` → zákazníci Professional a Complete dostávali `[object Object]` v PDF místo premium obsahu.

Po opravě: `premiumContent` je přidáno jako samostatné sekce za základní obsah smlouvy o dílo.

### Download route — trojitá validace tieru

```typescript
const resolvedTier = (draft.tier || (draft.payload as any).tier || 'basic');
const resolvedNotaryUpsell =
  draft.notaryUpsell === true ||
  Boolean((draft.payload as any).notaryUpsell) ||
  resolvedTier === 'professional' ||
  resolvedTier === 'complete';
```

Zákazník dostane správný obsah i tehdy, pokud Redis draft neobsahuje `notaryUpsell` flag (starší objednávky).
